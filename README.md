# Hermes Single-Agent

A minimal Docker setup for running a single [Hermes agent](https://github.com/NousResearch/hermes-agent) with **DeepSeek V4 Flash** as the LLM backend. Only needs Docker and a DeepSeek API key.

## Quick Start

```bash
# 1. Edit SOUL.md to define your agent's personality

# 2. Run setup — it will ask for API key, agent name, SOUL path,
#    then start the container and exec you in
bash setup.sh
```

Inside the container the virtual environment is activated automatically. Use:

```
hermes -p <agent_name> chat       # interactive chat
hermes -p <agent_name> shell      # interactive shell with tools
hermes -p <agent_name> run <cmd>  # single command
```

## Prerequisites

- A **Linux server** (or VM) that you can SSH into — all commands run on the server
- **Docker** with `docker compose` plugin installed on that server
- **OpenSSH server** — the agent uses SSH back to the host to execute commands
  ```bash
  sudo apt update && sudo apt install -y openssh-server
  ```
- **DeepSeek API key** — get one from [platform.deepseek.com](https://platform.deepseek.com)

## Configuration

All settings are stored in `.env` (auto-generated by `setup.sh`):

| Variable | Default | Description |
|---|---|---|
| `DEEPSEEK_API_KEY` | — | DeepSeek API key (required) |
| `AGENT_NAME` | `kaguya` | Hermes profile / agent name |
| `SOUL_PATH` | — | Path to your SOUL.md file |
| `API_SERVER_KEY` | auto-generated | Key for the internal API gateway |
| `PORT` | `8642` | API server port |

### Agent Name

The agent name doubles as the Hermes **profile name**. Change it in `.env`:

```
AGENT_NAME=my-agent
```

Then restart the container: `docker compose up -d`

### SOUL.md

This file defines your agent's personality (system prompt). Edit it before running `setup.sh`, or modify it later and restart:

```
docker compose restart
```

To use a different SOUL.md file, update `SOUL_PATH` in `.env` and restart.

## Project Structure

```
.
├── setup.sh              # Interactive setup wizard (run this)
├── custom-init.sh         # Container startup hook (cont-init.d, runs before system init)
├── docker-compose.yml    # Service definition
├── .env                  # Configuration (auto-generated, gitignored)
├── SOUL.md               # Agent personality definition (edit this)
├── templates/
│   ├── config.yaml       # Hermes config template (DeepSeek V4 Flash preset)
│   ├── global.env        # Global environment template
│   └── profile.env       # Profile environment template
├── README.md             # This file
├── README-CH.md          # Chinese documentation
└── .gitignore
```

## How It Works

1. `setup.sh` collects your config, writes `.env`, and starts the container via `docker compose up -d`
2. On container start, `custom-init.sh` runs (as a cont-init.d hook): renders config templates, creates the Hermes profile, and configures MCP conditionally
3. After setup, you are `exec`'d into the container with the Hermes venv pre-activated
4. Use `hermes -p <agent_name> ...` commands inside the container
